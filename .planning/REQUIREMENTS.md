# Requirements: Emily Idle

**Defined:** 2026-01-25
**Core Value:** A satisfying watch-collecting idle loop that saves reliably and stays pleasant to play and maintain.

## v2.1 Requirements

Requirements for this milestone (v2.1 Onboarding & UX).

### Guidance & Help

- [ ] **GUIDE-01**: User can open a Help/Glossary entry point from any tab.
- [ ] **GUIDE-02**: User can view contextual explanations for currencies and jargon at the point of use.
- [ ] **GUIDE-03**: User can see explanations for enjoyment gates vs cash spend and nostalgia unlock order.

### Clarity & Progress

- [ ] **CLAR-01**: User can see why a tab/system is locked and the unlock condition.
- [ ] **CLAR-02**: Catalog empty states explain what the panel is for and provide one clear next action.
- [ ] **CLAR-03**: User can see a basic rate breakdown (base + modifiers) for enjoyment and cash.
- [ ] **CLAR-04**: User can see progress feedback toward next unlocks (progress bar + "next unlock" callout).

### Prestige UX

- [ ] **PRES-01**: Before prestiging, user sees a clear lose/keep/gain summary with a safe confirmation.
- [ ] **PRES-02**: After prestige, user sees a re-onboarding summary and a recommended next action.

### UI Polish

- [ ] **POLISH-01**: Visual hierarchy improved for key stats and CTAs (typography/spacing emphasis).
- [ ] **POLISH-02**: Card/layout styling is consistent across tabs with reduced clutter.
- [ ] **POLISH-03**: Color/contrast improvements applied for readability and accessibility.
- [ ] **POLISH-04**: Micro-interactions added for primary buttons and progress transitions.
- [ ] **POLISH-05**: Consistent iconography used for locks/help/prestige cues.

## v2 Requirements

Deferred to a future milestone.

### Guidance & Help

- **GUIDE-04**: Onboarding steps resolve only after the player completes each action (action-gated onboarding).
- **GUIDE-05**: User can enable an "Explain like I'm new" mode to show extra guidance.
- **GUIDE-06**: User sees a post-change explanation panel when rates or multipliers shift.

### Progression Insights

- **INSIGHT-01**: User can view a prestige planner that compares rewards now vs later.
- **INSIGHT-02**: User receives adaptive hints when progress stalls.

### Meta Progression

- **META-01**: User can view a meta-loop map of collection -> workshop -> maison -> nostalgia.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Unskippable long tutorial | Blocks the idle loop and increases early churn. |
| Toast/popups for every unlock | Notification fatigue; interrupts play. |
| Hover-only explanations | Poor for touch/mobile; low discoverability. |

<task type="auto">
  <name>Task 1: Scaffold Vite + TypeScript app</name>
  <files>package.json, index.html, vite.config.*, tsconfig.json, src/*</files>
  <action>
    Create a Vite project suitable for a small browser game.

    Requirements:
    - Use Vite + TypeScript.
    - Prefer a minimal template to keep bundle small; if using React, keep it light and avoid additional state libs.
    - Ensure `npm run dev` starts successfully and serves a page.

    Avoid:
    - Adding extra frameworks or state management libraries.
    - Adding backend/server code; this should remain a static-site style frontend.
  </action>
  <verify>
    - Run: `npm install`
    - Run: `npm run dev` (server starts)
    - Run: `npm run build` (build succeeds)
  </verify>
  <done>
    Vite app boots locally; build succeeds; repo has standard Vite structure.
  </done>
</task>

<task type="auto">
  <name>Task 2: Add baseline tooling (format/lint/typecheck)</name>
  <files>package.json, .prettierrc*, .prettierignore, eslint.config.* or .eslintrc.*, tsconfig.json</files>
  <action>
    Add minimal, low-friction formatting/lint/typecheck scripts.

    Requirements:
    - Add Prettier with scripts `format` and `format:check`.
    - Add ESLint with a basic config appropriate for the chosen template.
    - Ensure ESLint and Prettier do not conflict (disable conflicting rules via `eslint-config-prettier`).
    - Add `typecheck` script (`tsc --noEmit`) if TypeScript is used.

    Avoid:
    - Running Prettier through ESLint rules unless absolutely necessary.
    - Overly strict lint rule sets that slow iteration in early phases.
  </action>
  <verify>
    - Run: `npm run format:check`
    - Run: `npm run lint` (or equivalent)
    - Run: `npm run typecheck`
  </verify>
  <done>
    Formatting, linting, and typechecking run cleanly on the scaffold.
  </done>
</task>
